# 📋 [냉털쿡] 기능 및 비기능 요구사항 명세서

본 요구사항 명세서는 AI 기반 냉장고 재료 레시피 추천 서비스 **'냉털쿡(자취생 요리 도우미 앱)'**의 핵심 기능과 비기능 성능 기준을 정의하여, 성공적인 애플리케이션 개발의 기준점을 제공합니다.

---

## 1. 프로젝트 개요

### 1.1 프로젝트 정의
* **프로젝트명**: 냉털쿡 (AI 기반 냉장고 재료 레시피 추천 앱)
* **목표**: 자취생 및 1인 가구가 냉장고 속에 남아있는 재료를 낭비하지 않고 쉽고 빠르게 맛있는 요리를 할 수 있도록 돕는 모바일 최적화 웹/앱 서비스입니다.
* **핵심 가치**: 
  * **식재료 절감**: 냉장고 속 남은 재료(냉털)의 적극적 활용
  * **시간 단축**: 자취생을 위한 초간단 요리법 제공 및 3초 이내 초고속 레시피 추천
  * **가계 도움**: 부족한 재료만 골라 마트에서 구매할 수 있도록 장보기 목록 연동

### 1.2 주요 타겟 고객
* 냉장고에 식재료가 애매하게 남아 처치가 곤란한 자취생
* 매번 "오늘 뭐 먹지?"를 고민하며 요리 난이도와 조리 시간에 민감한 1인 가구
* 요리 초보자로서 상세한 조리 단계와 정확한 재료 계량이 필요한 이용자

## 2. 기능 요구사항 (Functional Requirements)

기능 요구사항은 시스템이 반드시 수행해야 하는 행동과 동작을 기술합니다.

| 요구사항 ID | 요구사항명 | 상세 내용 및 처리 흐름 | 중요도 |
| :--- | :--- | :--- | :---: |
| **FR-01** | 냉장고 재료 직접 입력 | - 사용자는 텍스트 검색 창을 통해 냉장고 속 재료를 검색하여 추가할 수 있다.<br>- 입력된 재료는 화면 하단 '내가 입력한 재료' 리스트에 뱃지(Badge) 형태로 실시간 표시된다.<br>- 각 재료 오른쪽의 'X' 버튼을 눌러 개별 삭제하거나 '전체 삭제'를 할 수 있다. | **상** |
| **FR-02** | 재료 사진 업로드 및 음성 검색 | - 사용자는 모바일 카메라로 냉장고 내부나 식재료 사진을 촬영 또는 갤러리에서 업로드할 수 있으며, AI 이미지 인식을 통해 자동으로 재료 목록에 등록된다.<br>- 사용자는 검색창 옆 마이크 아이콘을 터치하고 음성으로 재료명을 말하여 손쉽게 재료를 검색하고 추가할 수 있다. | **중** |
| **FR-03** | AI 기반 요리 레시피 추천 | - 사용자가 선택한 재료 목록을 기반으로 인공지능이 최적의 매칭 레시피를 분석하여 다수의 요리를 추천한다.<br>- 추천 결과 화면에 보유 재료와 부족한 재료를 명확하게 시각화하여 매칭률을 보여준다. | **상** |
| **FR-04** | 부족한 재료 목록 제공 및 장보기 연동 | - 추천된 요리 카드의 하단에 해당 요리를 완성하기 위해 현재 보유하지 않은 '부족한 재료' 목록을 텍스트로 노출한다.<br>*(예: 김치볶음밥 추천 시 "부족한 재료: 쪽파")*<br>- 사용자는 요약 카드에서 부족한 재료를 한 번의 터치로 자신의 '장보기 목록'에 즉시 추가할 수 있는 편의 기능을 제공받는다. | **상** |
| **FR-05** | 레시피 상세 확인 | - 추천된 요리 카드를 클릭하면 상세 조리법으로 이동한다.<br>- 요리의 인분 정보(예: 1~2인분), 필요한 전체 재료 및 상세 계량 정보(컵, 스푼, g 단위)를 제공한다.<br>- 단계별 요리 순서(1, 2, 3...)를 텍스트와 시각적 요소로 명확히 표시한다. | **상** |
| **FR-06** | 조리 시간 및 난이도 표시 | - 모든 레시피 요약 카드 및 상세 화면에 예상 '조리 시간'(분 단위)과 '요리 난이도'(쉬움/중간/어려움)를 배지 형태로 노출한다. | **상** |
| **FR-07** | 레시피 즐겨찾기 (북마크) | - 사용자는 레시피 상세 페이지 및 추천 목록에서 하트(또는 별) 아이콘을 눌러 원하는 레시피를 보관함에 저장할 수 있다.<br>- '즐겨찾기' 탭에서 보관된 레시피 목록을 일괄 조회 및 편집(삭제)할 수 있다. | **상** |
| **FR-08** | 부족한 재료 장보기 목록 연동 | - 레시피 상세 화면 하단에 '재료 추가' 및 '재료 구매' 버튼을 제공한다.<br>- '재료 구매' 클릭 시 부족한 재료를 외부 쇼핑몰 장보기 카트에 담거나 내 장보기 메모 리스트에 자동 추가한다. | **중** |
| **FR-09** | 선호 음식 종류 카테고리 필터 | - 레시피 추천 요청 전, 사용자는 원하는 음식 종류(한식, 중식, 양식 등) 카테고리를 필터링하여 맞춤형 결과를 얻을 수 있다. | **중** |
| **FR-10** | 최근 검색 및 입력 기록 관리 | - 시스템은 사용자가 이전에 입력했거나 검색한 재료 및 레시피 로그를 데이터베이스(또는 LocalStorage)에 보관하여 재입력 편의성을 돕는다. | **하** |

## 3. 비기능 요구사항 (Non-Functional Requirements)

비기능 요구사항은 시스템의 성능, 사용성, 신뢰성, 보안성 등의 기술적 제약조건을 기술합니다.

### 3.1 성능 (Performance)
* **NFR-01: 초고속 레시피 추천 처리**
  * 사용자가 '레시피 추천받기' 버튼을 클릭한 시점부터 결과 화면에 추천 카드 리스트가 로드되는 시간은 **평균 3초 이내**여야 한다.
  * AI 추론 및 데이터베이스 조회 쿼리를 최적화하여 동시 접속자가 몰려도 이 응답 시간을 유지하도록 한다.

### 3.2 사용성 (Usability)
* **NFR-02: 스마트폰 최적화 반응형 레이아웃**
  * 모바일(스마트폰) 환경에서 화면 터치 반경, 폰트 크기(최소 12px 이상), 버튼 크기(최소 44x44px 이상)가 모바일 가이드라인을 준수해야 한다.
  * 스크롤 지연이 없어야 하며, 한 손 조작이 용이하도록 하단 탭 바(Bottom Navigation Bar) 형태의 네비게이션 구조를 적용한다.

### 3.3 저장성 및 데이터 관리 (Storage & Persistence)
* **NFR-03: 데이터 영속성 유지**
  * 사용자가 입력한 냉장고 재료 정보, 즐겨찾기 목록, 최근 검색 기록은 애플리케이션 프로세스가 종료되거나 웹 브라우저가 닫힌 후에도 유지되어야 한다.
  * 로그인 유저의 경우 서버 DB에 영구 보관되며, 비로그인 세션의 경우 웹 브라우저의 `LocalStorage` 또는 `IndexedDB`를 활용해 기기에 저장한다.

### 3.4 보안 및 접근 제어 (Security)
* **NFR-04: 개인정보 보호 및 안전한 로그인**
  * 사용자의 로그인 비밀번호는 단방향 암호화 알고리즘(SHA-256 / bcrypt 등)으로 해싱되어 데이터베이스에 저장되어야 한다.
  * 세션은 JWT(JSON Web Token) 방식을 사용하여 안전하고 무상태(Stateless)인 인증 구조를 확립한다.
